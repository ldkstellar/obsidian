코틀린 컴파일러는 첫번째로 코틀린 코드를 먼저 컴파일한다.
네, 맞습니다. 코틀린과 자바 코드가 섞여 있는 프로젝트를 빌드할 때, 빌드 도구는 일반적으로 **코틀린 코드를 먼저 컴파일하고, 그 다음에 남은 자바 코드를 컴파일**합니다.

### 단계별 컴파일 과정

1. **코틀린 코드 컴파일**: 빌드 도구(예: Gradle)가 먼저 **코틀린 컴파일러**를 실행해 프로젝트 내의 모든 `.kt` 파일을 JVM 바이트코드로 변환합니다. 이 과정에서 코틀린 코드가 자바 클래스를 참조하면, 컴파일러는 이미 존재하는 자바 파일의 정보를 읽어서 처리합니다.
    
2. **자바 코드 컴파일**: 코틀린 컴파일이 완료된 후, 빌드 도구는 **자바 컴파일러**를 실행해 모든 `.java` 파일을 컴파일합니다. 이때 자바 컴파일러는 코틀린 컴파일러가 이미 생성한 바이트코드(`*.class` 파일)를 참조하여, 자바 코드가 코틀린 코드를 문제없이 호출할 수 있도록 합니다.

순수 코틀린보다는 느리겠네 맞다.
