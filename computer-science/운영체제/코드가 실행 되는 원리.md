- 기계어파일이 메모리에 로드
- pcb(메모리 공간)에 프로세스 정보저장 페이지 테이블이 저장이 된다. 가상주소와 물리주소 (맵핑 배열형태로 설계됨)인덱스는 페이지 번호
- 페이지테이블 베이스 레지스터(ptbr) 에 페이지 테이블 시작 주소 저장. 빠르게 접근 
- 프로세스가 바뀔때마다 레지스터에 페이지 테이블 시작주소가 로드되고 해제된다.

### 변수읽기
기계어로 가상주소 생성
가상주소를 페이지번호와 오프셋 두부분으로 나눈다.
페이지 번호로 레지스터에 접근하여 페이지 테이블에있는 물리 주소로 접근 

### 변수쓰기
기계어로 가상주소 생성
가상주소를 페이지번호와 오프셋 두부분으로 나눈다.
페이지 번호로 레지스터에 접근하여 페이지 테이블에 물리메모리주소 맵핑

### 잘못된 접근

- 기계어로  잘못된 가상주소 생성
- 가상주소를 페이지번호와 오프셋 두분으로 나눈다.
- 페잊 번호로 레시스터에 접근하여 페이지 테이블에 물리 메모리 주소 맵핑
- 존재하지 않는 페이지 번호  결론 잘못된 메모리 접근


프로그램이 시작하면 운영체제에게 시스템 콜을 한다.

#### **OS가 바로 페이지 해제하지 않는 경우**

- 단순히 SP만 이동 → 해당 페이지 여전히 물리 메모리와 매핑된 상태
    
- 페이지 테이블 **변경 없음**
    
- 장점: 다시 접근 시 페이지 폴트가 없으므로 빠름
    
- 단점: 사용하지 않는 페이지가 계속 메모리 점유
    

  

#### **(B) OS가 스택 페이지를 해제하는 경우 (Sparse/Guard Page)**

- 특정 OS/환경에서는 스택의 상위 페이지를 **자동 언맵(unmap)** 할 수 있음
    
- 이때 페이지 테이블의 해당 엔트리 **Invalid** 또는 **Not Present**로 표시
    
- 이후 접근하면 페이지 폴트 → 다시 할당
