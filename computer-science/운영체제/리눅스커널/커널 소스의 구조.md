- arch: 특정 한 아키텍처에 따라 컴파일 되는 영역
- include: 커널코드 빌드에 필요한 헤더파일 모음 영역
- Documentation: 커널에 대한 설명이 써져 있다.
- kernal: 커널 소스코드 irq폴더에 인터럽트관련한 코드 sched: 스케줄 관련 코드 locking: 동기화 맞춰주기위한 코드 power: 전원관리하는 코드 printk: 커널 프린트 할때 사용 mm: 메모리 관리하는 소스 가상메모리 페이징관련 코드 driver: 장치관리 코드 fs: 파일 관리 코드 

### 바이너리 유틸리티
오브젝트 포맷의 파일을 조작할 수 있는 프로그램.

- objdump:라이브러리, ELF파일을 어셈블리어로 출력
- as: 어셈블러: 어셈블리어를 기계어로 해석
- ld:링커
- aaddr2line:주소를 파일과 라인으로 출력
- nm: 오브젝트 파일의 심벌을 출력
- readelf:elf 파일을 내용을 출력

![[Pasted image 20250621204700.png]]

```bash
objdump -d vmlinux #리눅스 파일구성을 보는 방법 헤더를 보는 방법
objdump -x vmlinux #어셈블리어 코드로 보는 방법
objdump --start-address=0xffffffc080d85310 --stop-address=0xffffffc080d85458 -d vmlinux
#특정함수를 어셈블리어코드를 보는법 시작함수 종료되는 함수
objdump --start-address=0xffffffc080d85310 --stop-address=0xffffffc080d85458 -d vmlinux > a.s #어셈블리코드로 변환해서 보기
```

 Linux 시스템에서 실행 파일, 공유 라이브러리, 오브젝트 파일 등에 주로 사용되는 표준 바이너리파일은 elf이다.
