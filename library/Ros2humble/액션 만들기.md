[[액션을 이해]]

1. 액션 정의하기
2.  .action 파일에 정의 된다.

```.action
# Request
---
# Result
---
# Feedback
```
--- 이건로 요청을 분리한다.

- request 메세지는 액션 클라이언트에서 액션서버로 전송되며  새로운 목표를 시작한다.
- result 메세지는 액션 서버에서 액션 클라이언트로 전송되며 목표가 끝났을 때 전송된다. 
- 피드백 메세지는 액센서버로 부터 액션 클라이언트로 목표에 대한 업데이트로

2. 액션 빌드하기 
	정의를  rosidl code generation을 통과 해야한다.

```cmake
find_package(rosidl_default_generators REQUIRED)

rosidl_generate_interfaces(${PROJECT_NAME}
  "action/Fibonacci.action"
)
```

##### rosidl은 무엇인가?
rosidl은 **ROS Interface Definition Language**의 약자로, ROS 2에서 메시지(msg), 서비스(srv), 액션(action) 파일을 기반으로 자동으로 코드를 생성하는 **인터페이스 코드 생성 시스템**입니다.

의존성이 필요하다.
```xml
<buildtool_depend>rosidl_default_generators</buildtool_depend>
<depend>action_msgs</depend>
<member_of_group>rosidl_interface_packages</member_of_group>
```
