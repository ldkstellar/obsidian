##### 뭐가 문제일까?
- std::map은 자동으로 정렬되는 컨테이너
- 그래서,요소 삽입/제거가 빈번할 경우 성능이 저하됨

```cpp
#include <iostream>

#include <string>

#include <unordered_map>

  

int main(int argc, char const *argv[]) {

std::unordered_map<std::string, int> scores;

scores["Nana"] = 60;

scores["Mocha"] = 70;

scores["Coco"] = 100;

for (auto it = scores.begin(); it != scores.end(); it++) {

std::cout << it->first << " " << it->second << std::endl;

}

// 정렬이 되지 않는다.

return 0;

}
```

std::unordered_map
- 키와 값의 쌍(pair)들을 저장
- 키는 중복 불가
- 자동으로 **정렬되지 않는** 컨테이너
	- 요소는 해쉬함수가 생성하는 색인(index)기반의 버킷(bucket)들로 구성됨
	- 해쉬맵(hashmap)이라고도 함
