- 컴파일러가 컴파일 도중에 각 템플릿 인스턴스에 대한 코드를 만들어 줌
	- 컴파일 타임은 비교적 느림
	- 템플릿 매개변수를 추가할수록 더느려짐
	- 하지만 런타임 속도는 더 빠를지도 모름 (함수 호출 안하니까
	- 실행 파일크기가 커지기 때문에 항상 그런것은 아님
	- c#과 Java도 어느정도 해당되는 말(ArrayList를 쓰지 말라는 것)
	- 자료형만 다른 중복 코드를 없애는 훌륭한 방법
	- 하지만 쓸모없는 템플릿 변형을 막을 방법이 없음
	
	```cpp
		add<int>();
		add<short>();
		add<unsigned int>();
	```
