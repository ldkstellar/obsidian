자바의 컴파일 속도가 빠른이유
- ✅ **C++**: 컴파일 시점에 모든 정의(헤더 포함)가 필요 → 링크 후 실행
- ✅ **Java**: 컴파일 시점에는 .class만 필요, 실제 클래스 로딩은 **런타임에 동적으로** 수행

각 클래스마다 정보가 있고 각 패키지마다 정보가 있기 때문이다. 자바는 런타임에 
하지만 기존 c++는 헤더파일 변경사항만 컴파일하지 않고 싹다 include한거 컴파일한다. 사

- c++17 까지도 여전히 c++ 표준으로 들어오지 않음
- 표준이 된다면 
	- 컴파일이 엄청 빨라짐
	- .cpp와 .h 파일로 (나눌 필요가 없음) 이건 컴파일 속도를 높이기 위한 것이었음
	- 자바의 패키지 처럼 작동
- 여전히 앞에 시련이 놓여 있음
	- .cpp와 .h 둘 다 있는 레거시 코드 처리
	- define을 너무 많이 사용 하고 있을 때
