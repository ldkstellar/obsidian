지금 여기서 문제는 tcp 타겟과  client 타겟의  전역으로 컴파일 디파인이 동일하다는 문제이다.
## 기존 Build System 문제점
전역으로 영향을 미치기 때문에 빌드 스케일 업이 불가능해 진다.
상호 참조나 숨겨진 의존성이 발생하기 쉽다.
결국 대부분의 개발자는 빌드 파일 관리 포기
타겟끼리 상호참조나 숨겨진 의존성이 발생된다.

## 해결방안

Modular 디자인 정의
- 의존성 그래프는 단방향 그래프여야 한다.
- 모듈은 고유한 역할을 담당
- 모듈은 쉽게 재사용

## 모듈이란
CMake에서 타겟에 해당한다.
