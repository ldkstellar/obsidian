동적 라이브러리는 **실행가능한 코드**이다.
정적 라이브러리는 **오브젝트 파일**의 모음이다.

##### 동적 라이브러리 특징

동적 라이브러리는 프로그램이 **실행될 때 또는 실행 중**에 메모리에 **로드** 된다.
동적 라이브러리는 여러 프로그램에 공유되어 사용될 수 있다.(저장공간을 절약 할 수 있다)
동적 라이브러리를 업데이트 하면, 해당 라이브러리를 사용하는 모든 프로그램이 새
버전의 라이브러리를 사용하게 된다.(호완성 및 보안 문제)
##### 정적 라이브러리 특징
정적 라이브러리는 컴파일 시간에 프로그램의 실행 파일에 포함된다.
정적 라이브러리는 여러 프로그램에 공유되지 않는다.
프로그램은 정적 라이브러리의 복사본을 갖기 때문에, 여러 프로그램이 동일한 라이브러리를 사용할 경우 저장공간 사용량이 증가한다.

##### 선호되는 라이브러리 종류는?

메모리와 저장공간의 비용은 매우 싸다.
보안상의 이유로 일부 운영체제에서는 정적라이브러리를 요구한다.
##### 라이브러리 타입 설정
동적 라이브러리는 shared, 정적 라이브러리는 static으로 설정하면된다.
